# 像素桌宠礼物：第一版项目计划

> 版本：v1.0  
> 制定日期：2026-07-13  
> 目标平台：iPhone / iPad  
> 开发环境：Linux  
> 发布方式：免费 PWA（Progressive Web App，可添加到主屏幕）

## 1. 项目结论

第一版实现为一个可安装到 iPhone/iPad 主屏幕的私人像素宠物 Web App。女朋友从 Safari 打开链接并“添加到主屏幕”后，可以像普通 App 一样启动；宠物在 App 打开期间会随机待机、走动、睡觉并响应点击、长按和拖动。

第一版不尝试让宠物永久悬浮在系统主屏幕、微信或其他 App 上。这是 iOS/iPadOS 的系统能力限制，不是本项目的技术目标。主屏幕图标保持静态，动态宠物在应用打开后显示。

### 1.1 第一版成功标准

最终交付应同时满足：

- 能在 Linux 上完成开发、构建和部署。
- 不需要 Mac、Xcode 或 Apple Developer 付费账号。
- 女朋友无需连接开发电脑，仅凭一个 HTTPS 链接即可安装。
- iPhone 和 iPad 上均可正常使用，竖屏为主要体验，横屏不崩溃。
- 应用打开时，宠物能连续播放动画并随机选择行为。
- 支持至少 3 种触摸互动。
- 第二次打开时能保留宠物名字、互动次数等本地数据。
- 首次加载后，即使暂时断网也可以继续打开主要内容。
- 不上传原始人物照片，不收集用户信息。

## 2. 产品范围

### 2.1 P0：必须完成

| 模块 | 第一版功能 | 验收要求 |
|---|---|---|
| 角色 | 根据照片制作一个透明背景像素小人 | 发型、衣着和至少一个个人特征可辨认 |
| 随机行为 | 待机、张望、行走、坐下、睡觉 | 连续观察 2 分钟，至少自然切换 3 种行为 |
| 点击互动 | 单击、连续点击、长按 | 三种输入产生不同且可见的反馈 |
| 拖动 | 将宠物拖到场景中的其他位置 | 松手后停在合法范围内，不移出屏幕 |
| 状态保存 | 名字、累计互动次数、首次打开日期 | 关闭并重新打开后数据仍存在 |
| PWA | 主屏幕图标、独立窗口、离线缓存 | 从 Safari 添加后可独立启动；断网可进入 |
| 适配 | iPhone/iPad 竖屏布局 | 无横向滚动条、按钮可触摸、刘海区域不遮挡 |
| 礼物内容 | 欢迎语和一段私人留言 | 首次进入时自然呈现，不影响后续使用 |

### 2.2 P1：第一版有余力再加入

- 根据本地时间切换早晨、白天、夜晚背景。
- 10～15 条随机气泡文字。
- 爱心值或亲密度，但不做复杂养成系统。
- 轻量音效及静音按钮。
- 纪念日当天触发专属动画和留言。
- 长时间无操作后提高“睡觉”行为概率。
- 隐藏彩蛋，例如连续点击特定物品后展示照片或文字。

### 2.3 P2：明确延后

- 原生 iOS/iPadOS App。
- 原生交互式小组件。
- App Store/TestFlight 发布。
- 账号、云端数据库和跨设备同步。
- AI 对话、语音识别或大模型 API。
- 推送通知。
- 多个角色、换装商城、复杂养成和联网社交。
- 在其他 App 上方悬浮。

延后这些功能的目的，是先确保礼物本身精致、稳定、有个人感，而不是把第一版变成一个长期产品。

## 3. 用户体验流程

### 3.1 安装流程

1. 你将最终链接或二维码发送给女朋友。
2. 她使用 Safari 打开链接。
3. 页面显示一页简短的礼物封面和“见见她”按钮。
4. 页面引导点击 Safari 分享按钮并选择“添加到主屏幕”。
5. 主屏幕出现自定义宠物图标和名称。
6. 此后从图标进入时，以独立窗口显示，不出现普通浏览器地址栏。

### 3.2 日常使用流程

1. 打开应用，读取本地存档和当前时间。
2. 显示场景，宠物播放入场或招手动画。
3. 进入随机行为状态机。
4. 用户点击、长按或拖动时，中断可中断的普通行为。
5. 播放互动动画、气泡文字或音效。
6. 更新互动次数/亲密度并保存。
7. 无操作后重新回到随机行为。

## 4. 技术方案

### 4.1 技术栈

| 技术 | 用途 | 你需要掌握的程度 |
|---|---|---|
| TypeScript | 业务逻辑、状态机、输入处理 | 能看懂类型、类、接口、枚举和函数即可 |
| Phaser 3 | 精灵动画、坐标、触摸/拖动、场景循环 | 重点学习 Scene、Sprite、Animation、Input |
| HTML/CSS | 页面容器、欢迎页、按钮、气泡和安全区 | 只需基础布局知识 |
| Vite | 本地开发服务器和生产构建 | 掌握 `dev`、`build`、`preview` 三个命令 |
| PWA 配置 | 主屏幕安装和离线缓存 | 主要由配置文件和 Service Worker 完成 |
| Git/GitHub Pages | 版本管理和免费 HTTPS 部署 | 沿用已有 Git 使用习惯 |
| Python（可选） | 检查/切分/缩放像素动画帧 | 可继续使用你熟悉的工具链 |

不引入 React、Vue、Node 后端或数据库。Node.js 只作为前端构建工具，不承担线上服务器功能。

### 4.2 总体架构

```text
PWA Shell
├── Welcome UI             首次打开、留言、安装提示
├── PetScene               Phaser 主场景
│   ├── PetController      宠物有限状态机
│   ├── ActionScheduler    加权随机动作调度
│   ├── Interaction        点击、长按、拖动识别
│   ├── BubbleController   对话气泡
│   └── AudioController    音效与静音
├── SaveStore              localStorage 本地存档
├── TimeRules              昼夜和纪念日规则
└── Service Worker         静态资源离线缓存
```

### 4.3 为什么适合你的背景

宠物行为层本质上是一个简化有限状态机，与机器人中常见的状态管理很接近：

```text
BOOT -> ENTER -> IDLE
                  ├── LOOK_AROUND
                  ├── WALK
                  ├── SIT
                  └── SLEEP

任意可中断状态 --用户输入--> REACT --动画结束--> IDLE
```

建议避免把所有行为写在一个大 `update()` 函数中。每种状态维护进入、更新、退出逻辑，随机调度器只决定“下一步做什么”，动画播放器只负责“怎么播放”。

### 4.4 随机行为策略

初始权重建议：

| 行为 | 权重 | 持续时间 |
|---|---:|---:|
| 待机呼吸 | 35 | 2～5 秒 |
| 左右张望 | 20 | 1.5～3 秒 |
| 行走 | 20 | 2～5 秒 |
| 坐下 | 15 | 3～6 秒 |
| 睡觉 | 10 | 5～12 秒 |

约束：

- 不连续选择两次完全相同的特殊行为。
- 宠物接近屏幕边界时只允许向屏幕内部移动。
- 用户刚互动后的 2～4 秒内不立刻睡觉。
- 夜间可以提高睡觉概率，但不强制。
- 随机数只影响下一个动作，不影响当前动画完整播放。
- 对用户输入设置短暂冷却，避免快速点击导致动画状态混乱。

## 5. 美术资产规范

### 5.1 照片输入

正式制作角色前，需要你提供：

- 1 张清晰正面或斜正面全身照。
- 1 张清晰半身近照，用于确认发型和五官特征。
- 1 张侧面或其他角度照片。
- 一套希望保留的标志性服装。
- 1～3 张希望接近的像素画风参考；如果没有，可先生成三种风格供选择。

照片仅用于角色设计。最终项目只放像素化透明 PNG，不放原始照片。

### 5.2 角色规格

- 推荐逻辑画布：`96 × 96 px`；人物实际占用约 `64 × 80 px`。
- 输出格式：透明背景 PNG 精灵表。
- 渲染方式：整数倍放大，关闭平滑插值，保持像素边缘清晰。
- 每一帧保持相同画布大小、脚底基准线和人物中心点。
- 左右行走如能镜像复用，则只绘制一侧；存在不对称发型/配饰时需检查镜像是否合理。
- 第一版固定一套服装和一个基础场景，避免资产数量失控。

### 5.3 第一批动画清单

| 动画 | 计划帧数 | 是否 P0 |
|---|---:|---:|
| 待机呼吸 | 4 | 是 |
| 行走 | 6～8 | 是 |
| 左右张望 | 4～6 | 是 |
| 坐下/起身 | 4～6 | 是 |
| 睡觉 | 6～8 | 是 |
| 点击惊讶 | 4～6 | 是 |
| 摸头开心 | 6～8 | 是 |
| 被连续点击后生气 | 6～8 | 是 |
| 挥手入场 | 6～8 | 可选 |
| 爱心彩蛋 | 6～10 | 可选 |

### 5.4 美术验收方法

在接入代码前先把精灵表单独预览：

- 正常速度和 2 倍慢速各播放一次。
- 检查脚底是否上下跳动、头发和衣服是否闪烁。
- 检查每个循环的首尾是否突变。
- 在浅色、深色背景上都检查透明边缘。
- 在 iPhone 实际尺寸下确认特征仍然可辨认。

## 7. 分阶段执行计划

总工作量按新手估算约 22～38 小时。AI 可以显著减少写代码时间，但真机适配、素材取舍和审美反馈仍需要你完成。建议按 3～4 个周末推进，不要连续完成全部阶段后才上手机测试。

### M0：环境与空项目（1～2 小时）

目标：在 Linux 浏览器中打开第一张可运行页面。

任务：

- 检查 Git、Node.js、npm 和浏览器环境。
- 创建 TypeScript + Vite 项目。
- 安装 Phaser 和 PWA 相关依赖。
- 建立 Git 仓库、README 和上述目录骨架。
- 加入最基础的自动格式化和类型检查。

我负责：生成项目骨架、配置文件、命令和解释。  
你负责：在 Linux 执行命令，把终端错误或成功页面反馈给我。  
验收：`npm run dev` 能打开页面；`npm run build` 成功且没有 TypeScript 错误。

### M1：无正式素材的技术原型（2～4 小时）

目标：先证明动画和触摸交互链路可行，不等待正式美术。

任务：

- 使用临时色块或占位小人加载 Phaser 场景。
- 播放一组循环动画。
- 完成单击和拖动。
- 完成 iPhone/iPad 尺寸响应式布局和安全区。
- 通过局域网或临时 HTTPS 方式在真机浏览器测试。

验收：Linux 浏览器和至少一台目标 iPhone/iPad 都能看到动画；触摸位置与角色位置一致；旋转屏幕后页面不崩溃。

### M2：人物风格与正式精灵资产（4～8 小时，可能分多轮）

目标：锁定角色外观并产出 P0 动画。

任务：

- 根据照片生成 2～3 个静态像素风格方案。
- 只在静态立绘通过后扩展动画，避免批量返工。
- 生成并整理各动作帧。
- 使用脚本检查画布、透明通道和帧尺寸。
- 输出精灵表及动画元数据。

我负责：生成风格候选、精灵素材和检查脚本。  
你负责：选择最像她且最适合礼物的方向，确认服装、发型和配色。  
验收：角色静态形象明确通过；P0 动画循环自然；所有精灵尺寸一致。

### M3：宠物行为状态机（3～5 小时）

目标：宠物在无人操作时表现得自然，而不是机械循环。

任务：

- 建立 `PetState`、`PetController` 和 `ActionScheduler`。
- 接入待机、张望、行走、坐下、睡觉。
- 实现权重、随机持续时间、边界约束和状态冷却。
- 增加可开关的调试面板，显示当前状态、剩余时间和随机选择结果。

验收：连续运行 10 分钟无卡死、无越界、无快速闪帧；调试面板中的状态转换符合预期。

### M4：触摸互动与本地存档（3～5 小时）

目标：完成礼物最核心的“她点它，它回应”体验。

任务：

- 区分单击、短时间连续点击、长按和拖动。
- 交互时安全中断普通行为，播放对应反应。
- 添加气泡文字和点击冷却。
- 使用 `localStorage` 保存名字、互动次数、静音选项等。
- 处理存档不存在、字段升级或损坏时的默认恢复。

验收：四种输入不会互相误触；快速点击不会让状态机锁死；刷新和重新打开后存档保留。

### M5：PWA、离线和免费部署（2～4 小时）

目标：让应用无需开发电脑即可安装和使用。

任务：

- 配置应用名称、主题色、主屏幕图标和启动方式。
- 缓存 HTML、JavaScript、精灵图、背景和基础音效。
- 部署到支持 HTTPS 的免费静态站点。
- 编写应用内安装引导。
- 使用 Safari 完成“添加到主屏幕”。

验收：主屏幕图标正确；从图标打开时为独立窗口；完成首次加载后关闭网络仍能打开主要场景。

### M6：礼物化与个性内容（3～6 小时）

目标：把技术 Demo 变成真正的礼物。

任务：

- 完成首次欢迎页和私人留言。
- 加入昼夜规则、纪念日或一个隐藏彩蛋。
- 调整背景、字体、气泡和动画节奏。
- 加入音效与静音开关；音频只在用户第一次交互后启用。
- 删除开发占位文本和不必要的调试界面。

验收：第一次打开有完整的礼物体验；日常再次打开不会重复冗长引导；无明显“开发样例”痕迹。

### M7：真机测试与发布冻结（2～4 小时）

目标：形成可送出的稳定版本。

任务：

- 在目标 iPhone 和 iPad 上逐项执行测试清单。
- 检查竖屏、横屏、锁屏恢复、后台恢复、断网和低电量模式。
- 检查第一次安装和已有存档升级。
- 修复高优先级问题，锁定 v1.0。
- 保存源码、生产构建、素材源文件和部署说明。

验收：所有 P0 用例通过；连续使用 20 分钟无严重问题；从零开始安装流程可由他人独立完成。

## 8. 测试计划

### 8.1 功能测试

- 每一种随机行为都能进入、完整播放并退出。
- 点击、连续点击、长按、拖动能被正确区分。
- 拖动到四个边缘不会让角色消失。
- 睡觉时点击能够正常唤醒。
- 连续快速点击 20 次不会卡死或多重叠加动画。
- 应用重载后存档正确；清除网站数据后能恢复默认状态。
- 时间规则在边界时刻不会重复触发彩蛋。

### 8.2 视觉测试

- iPhone 小屏、常规屏和 iPad 上角色比例合理。
- 刘海、灵动岛、底部 Home Indicator 不遮挡内容。
- 像素图没有模糊插值。
- 气泡不会超出屏幕或挡住主要人物特征。
- 浅色和深色背景中角色轮廓清晰。
- 横竖屏切换后角色仍位于有效区域。

### 8.3 生命周期和网络测试

- 打开应用后锁屏，再解锁恢复。
- 切换到其他 App 1 分钟后再回来。
- 关闭网络后从主屏幕启动。
- 发布新版本后能更新缓存，而不是永久停留在旧版本。
- 第一次加载中途断网时有可理解的错误提示。

### 8.4 性能基线

- 正常动画目标 60 FPS；较旧设备允许稳定 30 FPS。
- 避免在每一帧创建新对象或打印日志。
- 精灵表和背景在保证观感的前提下压缩。
- 不在主循环中访问 `localStorage`；只在状态变化或需要保存时写入。
- 页面隐藏时暂停非必要逻辑，恢复后重新校正时间。

## 9. 部署、隐私与备份

### 9.1 发布策略

- 使用静态托管，不需要服务器常驻进程。
- 每次发布前必须通过 `typecheck`、`build` 和基本真机冒烟测试。
- 使用版本标签：`v0.1-prototype`、`v0.5-interactions`、`v0.9-rc`、`v1.0-gift`。
- v1.0 后优先修复严重问题，不在送礼前临时加入大功能。

### 9.2 隐私边界

- 原始照片只用于生成素材，不提交到代码仓库和部署目录。
- 线上只包含最终像素图；仍需假设访问者可以下载这些公开静态资源。
- 不接入广告、统计 SDK、第三方登录和定位权限。
- 私人留言不要包含身份证号、家庭地址等敏感信息。
- 如果部署地址不希望被搜索，可使用难猜路径并禁止搜索引擎索引，但这不能等同于真正的访问控制。

- 原始照片与美术生成过程文件单独私密保存。
- 最终精灵表和场景素材。

## 10. 主要风险及应对

| 风险 | 影响 | 应对方式 |
|---|---|---|
| 把 PWA 误解为系统悬浮桌宠 | 最终体验与预期不符 | 项目开始即锁定“打开 App 后互动”的产品定义 |
| AI 生成动画帧人物不一致 | 角色闪烁、动作不可用 | 先锁定静态角色；小批量生成；脚本检查后人工修帧 |
| 只有 Linux，无法使用 iOS 模拟器 | 某些问题较晚发现 | 从 M1 开始每个阶段都用真机 Safari 测试 |
| iOS 音频自动播放限制 | 首次启动没有声音 | 第一次点击后初始化音频；始终保留静音使用路径 |
| PWA 缓存旧版本 | 发布修复后仍显示旧内容 | 明确缓存版本；更新时清理旧缓存；提供版本号页面 |
| 免费静态托管资源公开 | 私人素材泄露 | 不上传原始照片；只发布像素化结果；不放敏感信息 |
| 第一版功能膨胀 | 延期且难以收尾 | 只有 P0 阻塞发布，P1 随时可砍，P2 不进入 v1.0 |
| 点击/拖动手势冲突 | 互动体验混乱 | 使用移动阈值、按压时长和统一输入控制器判定 |
| 后台恢复后状态异常 | 动画快进或定时器混乱 | 监听页面可见性；后台暂停；回来后重新调度状态 |

## 11. 里程碑决策点

为了减少返工，每个决策点通过后再进入下一阶段：

1. **技术决策点**：M1 真机原型通过，确认 PWA 体验可以接受。
2. **角色决策点**：静态像素人物通过，才生成整套动画。
3. **交互决策点**：M4 的手势和状态机稳定，才添加彩蛋和音效。
4. **发布决策点**：P0 全部通过后冻结功能，只修缺陷。

## 12. 你与 AI 的后续协作方式

## 14. Definition of Done：v1.0 完成定义

只有满足以下全部条件，项目才视为可送出：

- P0 功能全部实现并通过验收。
- 目标 iPhone 和 iPad 至少各完成一次安装测试。
- 主屏幕图标、应用名称和欢迎页均为最终内容。
- 随机状态机运行 20 分钟无卡死、越界或严重闪帧。
- 四类输入不会造成状态损坏。
- 断网可启动，恢复网络后无需人工修复。
- 原始照片和敏感信息未进入部署包。
- 仓库存在 v1.0 标签，并保存恢复部署说明。
- 女朋友可以不接触开发电脑，独立从链接完成安装。
